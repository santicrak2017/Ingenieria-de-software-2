### → ¿Qué API elegiste y por qué?

Escogí el API de Pokémon, porque al ingresar a esta me pareció interesante la manera en que se restringe la cantidad de Pokémon que podremos tener, esto de cierta manera por default.

---

### → ¿Qué datos devuelve?

Solo deja devolver los datos que están en el mismo JSON, como pueden ser todos los datos, habilidades y número de Pokémon en específico. Aunque no deja ni crear o borrar algún dato que está en el JSON y te manda un error 404 al intentar hacer alguna de estas acciones.

---

### → ¿Usa token o no? ¿Qué tipo?

No usé token, aunque este es importante en un ambiente laboral para determinar qué fue lo que uno trabajó. En esta tarea como la única persona que trabaja soy yo, entonces no vi obligatorio usar algún tipo de token.

---

### → ¿Qué código de estado recibiste en cada request?

En la mayoría de estas me apareció como estado 200, en donde no tuve ningún problema para que me diera el resultado, pero al querer hacer un POST me da que no se puede hacer (error 404).

---

### → ¿Qué aprendiste diferente a JSONPlaceholder?

Que en este tipo de APIs no se puede hacer algo más que revisar la información que está en esta y nada más, además de saber que hay diferentes querys predeterminados como los de máximo 20 ítems en la página. En otros que miré adicional a este había otras formas de intentar sacar los ítems en específico.
